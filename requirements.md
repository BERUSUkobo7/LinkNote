Todo管理アプリ 要件定義書（最新版）
1. 概要

本アプリは、名前・URL・タグを持つTodo（リンク付きタスク）をスマートフォン上で簡単に管理するTodoアプリである。
スクロール可能な一覧、タグによるフィルタ、URLアクセス、並び替えなどをローカル環境のみで行えるように設計する。

2. 対象プラットフォーム・データ保存

スマートフォンアプリ（iOS / Android）

データはすべてスマホ端末内にローカル保存

オフラインでも全機能が利用可能

3. データモデル
項目名	型	必須	説明
id	string/int	◯	一意な識別子
name	string	◯	Todo名
url	string	△	関連URL
tags	string[]	△	0個以上のタグ
orderIndex	int	◯	並び順管理用インデックス
createdAt	datetime	◯	作成日時
updatedAt	datetime	◯	更新日時
4. UI / 画面要件
4.1 一覧画面
4.1.1 レイアウト

上部：アプリタイトル、タグフィルタUI

中央：スクロール可能なTodo一覧（縦スクロール）

右下：常時表示の「追加」ボタン（FAB）

4.1.2 Todoアイテム表示内容

名前（name）

URLボタン（押下でブラウザ or WebView を開く）

URL未設定時 → 無効化 or 非表示（実装時に決定）

4.1.3 Todoの操作（重要）

左スワイプ → 削除ボタンを表示

削除ボタンタップで確認ダイアログ表示 → OKで削除実行

右スワイプ → 編集ボタンを表示

編集ボタンタップで編集画面を開く

長押し → 並び替えモードに入り、ドラッグ＆ドロップで順番を変更

並び替え終了後、orderIndex を更新してローカル保存

4.2 追加 / 編集画面
4.2.1 表示される項目

追加/編集時のみ以下が入力可能：

項目	必須	説明
名前	◯	タイトル
URL	△	入力チェックあり
タグ	△	カンマ or スペース区切り、複数可能
4.2.2 ボタン

保存

キャンセル

5. 機能要件
5.1 一覧表示

ローカル保存されたTodoを orderIndex 順に表示

スクロールバー表示あり

タグフィルタで「特定タグのみ」または「すべて」を切り替え

5.2 追加

追加ボタン → 追加画面

入力内容を保存後、一覧に追加

5.3 編集（右スワイプ）

右スワイプ → 編集ボタン表示

編集画面で内容を変更

保存するとローカルデータ更新

5.4 削除（左スワイプ）

左スワイプ → 削除ボタン表示

削除確認ダイアログ → OK後削除

削除後は一覧から除外し、ローカルデータからも削除

5.5 並び替え（長押し）

長押し → 並び替えモード

ドラッグでTodoの順番を変更

変更後の順番をローカル保存

5.6 URLオープン

URLボタン押下でブラウザ or WebView表示

URL未設定時の挙動は実装時に調整

6. 非機能要件

オフライン完全対応

操作感はスマホ標準のUIUX（スワイプ・長押し操作）

大量のTodoでもスクロール・並び替えが快適に動作する
